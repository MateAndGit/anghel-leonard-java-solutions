# [Java] P49: 불변 클래스(Immutable Class) 작성법

P48에서 문자열의 불변성이 깨질 때의 위험성을 보았다면, 이번 P49에서는 우리가 직접 **절대 변하지 않는 안전한 클래스**를 만드는 정석적인 방법을 배웁니다.

---

## 1. 불변 클래스 작성을 위한 5대 원칙

불변 클래스는 객체가 생성된 후 그 상태를 평생 유지해야 합니다. 이를 위해 다음 요구사항을 반드시 준수해야 합니다.

1.  **클래스를 `final`로 선언**: 다른 클래스에서 상속받아 메서드를 오버라이딩(재정의)하여 동작을 변경하는 것을 막습니다.
2.  **모든 필드를 `private` 및 `final`로 선언**: 외부 노출을 차단하고, 생성자에서 딱 한 번만 초기화되도록 강제합니다.
3.  **public 생성자(또는 팩토리 메서드) 제공**: 모든 필드를 매개변수로 받아 초기화하는 수단이 필요합니다.
4.  **Getter 제공**: 필드 값을 읽을 수 있는 메서드를 제공합니다.
5.  **Setter 미노출**: 값을 변경하는 메서드(`setX`, `setY` 등)를 절대 만들지 않습니다.


---

## 2. 코드 분석: Point 클래스

제공해주신 `Point` 클래스는 위 5가지 조건을 모두 완벽하게 충족하는 **불변 객체**의 표본입니다.

- **`public final class Point`**: 상속 불가능.
- **`private final double x, y`**: 외부 수정 불가능하며 한 번 정해지면 고정됨.
- **`public Point(double x, double y)`**: 생성 시점에 값을 주입.
- **Getter만 존재**: 읽기 전용.

---

## 3. 왜 이렇게까지 '불변'에 집착하나요?

실무에서 불변 객체를 쓰면 얻는 이득이 엄청나기 때문입니다.

- **캐시(Cache)에 안전**: 값이 변하지 않으니 안심하고 재사용할 수 있습니다.
- **멀티스레드 안전(Thread-Safe)**: 여러 스레드가 동시에 `Point` 객체를 참조해도 값이 바뀔 리 없으므로 별도의 동기화 처리가 필요 없습니다.
- **부작용(Side Effect) 방지**: 어떤 메서드에 이 객체를 넘겨도 내부 데이터가 변조될 걱정이 없습니다.

---

## 4. 실무형 요약 보고서

| 항목 | 구현 방법 | 효과 |
| :--- | :--- | :--- |
| **상속 방지** | `final class` | 클래스 확장 및 로직 변조 차단 |
| **수정 방지** | `final field` | 메모리 할당 후 값 변경 불가 |
| **접근 제어** | `private field` | 캡슐화를 통한 직접 접근 차단 |
| **변경 수단** | **Setter 없음** | 객체의 상태 영속성 보장 |

---

## 5. 현업 개발자의 유지보수 팁 📝

### ⚠️ 주의: 참조 타입 필드가 있다면?
만약 `Point` 내부에 `List`나 `Date` 같은 가변(Mutable) 객체가 필드로 있다면, 단순히 `final`만 붙인다고 불변이 되지 않습니다. 이때는 **방어적 복사(Defensive Copy)** 기법을 써야 하는데, 이는 다음 장에서 다루게 될 핵심 내용입니다.

### 💡 Java 14+의 대안: Record
최신 자바에서는 위와 같은 코드를 일일이 짤 필요 없이 `record Point(double x, double y) {}` 한 줄로 이 모든 불변 로직을 자동 생성할 수 있습니다. 하지만 원리를 아는 것은 여전히 중요합니다.
