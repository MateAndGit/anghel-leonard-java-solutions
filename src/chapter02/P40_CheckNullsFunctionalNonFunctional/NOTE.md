# [Java] Numbers 클래스 분석: 안전한 리스트 연산과 Null 방어

이 클래스는 리스트 내의 숫자 데이터를 다룰 때 발생할 수 있는 Null 문제와 기본 타입 변환을 안전하게 처리하는 방법을 다룹니다.

---

## 1. 메서드별 상세 분석

### ① `sumIntegers`: 스트림을 이용한 안전한 합계 계산
- **원리**: 리스트가 `null`인지 먼저 체크한 후, 스트림 내부에서 `.filter(Objects::nonNull)`을 통해 요소 중 `null`인 값을 걸러냅니다.
- **특징**: `Integer`(객체)를 `int`(기본 타입)로 바꾸는 `mapToInt`를 사용하여 산술 연산 성능을 높였습니다.

### ② `integersContainsNulls`: Null 존재 여부 확인
- **원리**: 스트림의 `anyMatch`를 사용하여 리스트 안에 `null`이 하나라도 포함되어 있는지 검사합니다.
- **특징**: 데이터의 무결성을 검증할 때 매우 유용하며, `boolean` 값을 빠르게 반환합니다.

### ③ `evenIntegers`: 짝수 필터링 (전통적 방식)
- **원리**: `for-each` 루프를 돌며 짝수(`nr % 2 == 0`)를 찾아 새로운 리스트에 담습니다.
- **특징**: `Collections.EMPTY_LIST`를 반환하여 호출하는 쪽에서 `null` 체크를 하지 않아도 되게끔 배려하는 **방어적 프로그래밍**의 정석을 보여줍니다.

---

## 2. 핵심 개념: Null 방어 전략

자바에서 객체 리스트를 다룰 때는 항상 '리스트 자체가 null인 경우'와 '리스트 안의 알맹이가 null인 경우' 두 가지를 모두 고려해야 합니다.



---

## 3. 실무 활용 사례

### [사례 1: 통계 데이터 집계]
- DB에서 가져온 매출 데이터 리스트에 가끔 비어있는 값(`null`)이 섞여 있을 때, 시스템이 멈추지 않고 합계를 계산해야 하는 경우 `sumIntegers` 로직이 필수입니다.

### [사례 2: 데이터 유효성 검사]
- 사용자로부터 입력받은 숫자 배열에 누락된 항목이 있는지 확인할 때 `integersContainsNulls`를 사용하여 에러 메시지를 띄울 수 있습니다.

### [사례 3: 필터링 서비스]
- 쇼핑몰에서 "가격이 짝수인 상품만 보기"와 같은 필터 기능을 구현할 때 `evenIntegers`와 같은 필터링 로직이 사용됩니다.

---

## 4. 현업 개발자의 유지보수 팁 📝

### ⚠️ Objects.isNull() vs == null
`Objects.isNull(obj)`는 자바 8부터 도입되었습니다. `obj == null`과 기능은 같지만, 스트림 API 내에서 `filter(Objects::nonNull)`처럼 **메서드 참조**로 쓸 수 있어 가독성이 훨씬 좋아집니다.

### 💡 빈 리스트 반환의 미학
메서드 결과가 없을 때 `null`을 반환하면 이를 호출한 쪽은 매번 `if (result != null)`을 써야 합니다. `Collections.emptyList()`를 반환하는 습관은 팀원들에게 사랑받는 개발자가 되는 지름길입니다.
