# [Java] Strings 클래스 분석: 부분 문자열 포함 여부 확인 (Contains Substring)

문자열 검색은 거의 모든 자바 애플리케이션에서 사용되는 기능입니다. 이번 예제는 유연한 검색(정규식)과 빠른 검색(인덱스)의 차이를 보여줍니다.

---

## 1. 메서드별 상세 분석

### ① V1: matches()와 정규표현식
- **원리**: `Pattern.quote()`로 검색어를 안전하게 감싸고, 앞뒤에 `.*`(모든 문자)를 붙여 정규식 패턴을 만듭니다.
- **특징**: `(?i)` 플래그를 사용하여 **대소문자를 구분하지 않는(Case-insensitive)** 검색이 가능합니다. 하지만 정규식 엔진을 거쳐야 하므로 성능 오버헤드가 발생합니다.

### ② V2: indexOf() 방식 (추천)
- **원리**: 문자열 내부에서 해당 단어가 시작되는 첫 번째 위치(Index)를 찾습니다.
- **특징**: 단순히 포함 여부만 확인할 때는 가장 빠르고 효율적입니다. 결과값이 `-1`이 아니면 포함된 것으로 간주합니다.

---

## 2. 알고리즘 시각화: 문자열 매칭 과정

문자열 검색은 긴 텍스트 위에서 돋보기를 움직이며 패턴이 일치하는 지점을 찾는 과정과 같습니다.



---

## 3. 실무 활용 사례

### [사례 1: 스팸 필터링 및 금지어 검사]
- 게시판 글이나 채팅 메시지에 "광고", "도박" 같은 금지어가 포함되어 있는지 실시간으로 확인할 때 사용합니다.
- `if (message.contains("광고")) { ... }`

### [사례 2: 검색 기능 구현]
- 사용자가 입력한 키워드가 상품명이나 게시글 제목에 포함되어 있는지 필터링할 때 기초 로직으로 쓰입니다. 대소문자 구분 없는 검색이 필요할 때 `V1` 방식이 유용합니다.

### [사례 3: 로그 분석]
- 서버 로그 파일에서 "ERROR"나 "Exception"이라는 단어가 포함된 줄만 골라내어 모니터링 시스템에 알람을 보낼 때 활용됩니다.

---

## 4. 실무형 요약 보고서

| 방식 | 성능(속도) | 유연성 | 대소문자 무시 | 추천 상황 |
| :--- | :---: | :---: | :---: | :--- |
| **V1 (matches)** | 낮음 | **최상** | 가능 (`(?i)`) | 복잡한 패턴 검색 시 |
| **V2 (indexOf)** | **최상** | 낮음 | 직접 구현 필요 | 단순 포함 여부 확인 시 |

---

## 5. 현업 개발자의 유지보수 팁 📝

### ⚠️ `String.contains()`의 정체
실제로 실무에서는 `text.contains(subtext)`를 가장 많이 씁니다. 자바 내부 코드를 뜯어보면 이 메서드 역시 내부적으로 `indexOf(subtext.toString()) > -1`을 호출하고 있습니다. 즉, **V2**와 동일한 원리입니다.

### 💡 대소문자 구분 없이 indexOf 쓰고 싶을 때
정규식(`V1`)은 느리기 때문에, 성능이 중요하다면 두 문자열을 모두 소문자로 바꾸어 비교하는 방식을 주로 씁니다.
```java
return text.toLowerCase().contains(subtext.toLowerCase());
