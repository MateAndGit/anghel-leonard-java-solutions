# [Java] Strings 클래스 분석: 부분 문자열 출현 횟수 세기

문자열 안에 특정 단어나 패턴이 몇 번 반복되는지 확인하는 세 가지 방식을 다룹니다. 특히 '중첩(Overlapping)' 처리 방식의 차이를 이해하는 것이 핵심입니다.

---

## 1. 메서드별 상세 분석

### ① V1: indexOf() 루프 방식 (중첩 미허용)
- **원리**: `indexOf(toFind, position)`를 사용하여 단어를 찾은 뒤, `position`을 **찾은 단어의 길이만큼** 건너뜁니다.
- **특징**: "AAA"에서 "AA"를 찾을 때, 앞의 "AA"만 세고 뒤는 건너뜁니다 (결과: 1). 실무에서 가장 일반적인 '단어 개수 세기' 방식입니다.

### ② V2: split() 트릭 방식
- **원리**: 찾으려는 단어를 구분자로 삼아 문자열을 쪼갠 뒤, 생성된 배열의 길이에서 1을 뺍니다.
- **특징**: 코드가 한 줄로 매우 짧지만, 내부적으로 배열을 생성하므로 메모리 소모가 크고 성능이 가장 낮습니다. (추천하지 않음)

### ③ V3: Matcher & start() 방식 (중첩 허용)
- **원리**: 정규표현식 `Matcher`를 사용하며, 다음 검색 위치를 `matcher.start() + 1`로 설정합니다.
- **특징**: **중첩된 발생**을 잡아낼 수 있습니다. "AAA"에서 "AA"를 찾을 때 0번 인덱스의 "AA"와 1번 인덱스의 "AA"를 모두 카운트합니다 (결과: 2).

---

## 3. 실무 활용 사례

### [사례 1: 로그 파일 패턴 분석]
- 서버 로그에서 특정 에러 코드나 사용자 ID가 몇 번 등장했는지 집계하여 대시보드에 표시할 때 사용합니다. (보통 V1 방식 선호)

### [사례 2: DNA 서열 분석]
- "ATGC..." 와 같은 염기 서열 데이터에서 특정 유전자 패턴이 몇 번 나타나는지 분석할 때 사용합니다. 이때는 생물학적 의미에 따라 V3처럼 중첩된 패턴을 찾아야 할 수도 있습니다.

### [사례 3: 검색 엔진의 키워드 강조]
- 사용자가 검색한 키워드가 본문에 몇 번 포함되었는지 계산하여 상단에 "총 N건의 결과가 발견되었습니다"라고 표시할 때 활용합니다.

## 4. 실무형 요약 보고서

| 방식 | 성능 | 중첩 처리 | 특징 | 추천 상황 |
| :--- | :---: | :---: | :--- | :--- |
| **V1 (indexOf)** | **최상** | 미허용 | 빠르고 표준적임 | 일반적인 단어 개수 세기 |
| **V2 (split)** | 낮음 | 미허용 | 구현이 매우 쉬움 | 아주 짧은 문자열 간단 처리 |
| **V3 (Matcher)** | 보통 | **허용** | 유연한 패턴 검색 | 중첩된 패턴까지 찾아야 할 때 |

## 5. 현업 개발자의 유지보수 팁 📝

### ⚠️ Pattern.quote()의 중요성
`split`이나 `Pattern.compile`을 쓸 때 찾으려는 문자열에 정규식 특수문자(`*`, `+`, `?` 등)가 포함되어 있으면 프로그램이 터질 수 있습니다. `Pattern.quote(toFind)`는 이를 안전한 일반 문자로 바꿔주는 아주 중요한 장치입니다.

### 💡 성능이 정말 중요하다면?
엄청나게 긴 텍스트(수십 MB 이상)에서 특정 패턴을 반복해서 찾아야 한다면, 자바 기본 API보다 더 빠른 **보이어-무어(Boyer-Moore)** 혹은 **KMP** 알고리즘 구현체를 고려하는 것이 좋습니다.
