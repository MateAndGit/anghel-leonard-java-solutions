# 🔍 Strings 클래스 메서드 상세 분석

이 클래스는 문자열에서 중복 문자의 개수를 세는 4가지 버전을 제공합니다.

---

### 1. countDuplicateCharactersV1 (기본형)
가장 표준적인 자바 8 이전 스타일의 접근 방식입니다.

* **동작 방식**: `for` 루프를 돌며 `str.charAt(i)`로 문자를 하나씩 가져옵니다.
* **핵심 기술**: `Map.compute()` 메서드를 사용합니다.
    * `v == null`: 처음 발견한 문자면 1을 저장.
    * `++v`: 이미 있다면 기존 값에 1을 더함.
* **한계**: '😊'와 같은 이모지는 내부적으로 2개의 char로 저장되는데, 이를 각각 별개의 문자로 인식해버리는 단점이 있습니다.

---

### 2. countDuplicateCharactersVCP1 (이모지 대응 루프)
V1의 단점을 보완하기 위해 **Code Point(유니코드 번호)** 개념을 도입했습니다.

* **동작 방식**: `str.codePointAt(i)`를 사용하여 실제 문자의 고유 번호를 가져옵니다.
* **Surrogate Pair 처리**:
    ```java
    if(Character.charCount(cp) == 2) { i++; }
    ```
  이 부분은 이모지처럼 2칸을 차지하는 문자를 만났을 때, 인덱스(`i`)를 한 칸 더 건너뛰어 다음 문자로 정확히 이동하게 합니다.
* **결과**: 이모지도 깨지지 않고 하나의 문자로 정확히 카운트합니다.

---

### 3. countDuplicateCharactersV2 (Stream 활용)
자바 8의 **Stream API**를 사용하여 코드를 획기적으로 줄인 버전입니다.

* **동작 방식**:
    1.  `str.chars()`: 문자열을 정수 스트림으로 변환.
    2.  `.mapToObj(c -> (char) c)`: 숫자를 다시 문자(`char`) 객체로 변환.
    3.  `.collect(Collectors.groupingBy(...))`: 같은 문자끼리 그룹을 묶고 개수를 셈.
* **장점**: 코드가 매우 간결하고 읽기 쉽습니다. (선언적 프로그래밍)

---

### 4. countDuplicateCharactersVCP2 (가장 완벽한 방식)
Stream API와 Code Point 처리를 결합한 **최종 진화형**입니다.

* **동작 방식**:
    1.  `str.codePoints()`: 이모지까지 포함한 정확한 유니코드 스트림을 생성.
    2.  `Character.toChars(c)`: 유니코드 번호를 실제 문자열로 변환.
    3.  이후 V2와 동일하게 그룹화하여 카운트.
* **결론**: 실무에서 다국어나 이모지를 포함한 문자열을 처리해야 한다면 **이 방식이 가장 안전하고 권장되는 방식**입니다.

---

### 🏢 실무에서의 활용 사례 (VCP2 기준)

#### 1. SNS 및 채팅 서비스의 "금지어/중복 문자" 필터링
사용자가 아이디를 만들거나 댓글을 쓸 때, 특정 문자를 너무 많이 도배하는 것을 막아야 할 때가 있습니다.
* **예시**: "ㅋㅋㅋㅋㅋㅋㅋ"를 10번 이상 쓰지 못하게 제한하거나, 특정 이모지의 사용 빈도를 체크할 때 사용합니다.
* **이유**: `char` 방식(V1, V2)으로 이모지를 체크하면 이모지 하나가 두 글자로 인식되어 필터링 로직이 꼬일 수 있기 때문입니다.

#### 2. 텍스트 분석 및 태그 클라우드 (Text Mining)
고객의 리뷰나 설문조사 결과를 분석할 때, 어떤 단어나 문자가 가장 많이 언급되었는지 통계를 낼 때 기초 자료로 활용됩니다.
* **예시**: "이 제품 너무 좋아요 😊😊"라는 리뷰에서 '😊'가 2번 나왔다는 것을 정확히 파악하여 감성 분석의 지표로 삼습니다.

#### 3. 문자열 압축 알고리즘 (Huffman Coding 등)
데이터를 전송할 때 용량을 줄이기 위해, 가장 많이 등장하는 문자에 짧은 코드를 부여하는 압축 알고리즘의 **사전 단계**로 쓰입니다.
* **절차**: 각 문자의 빈도수를 먼저 파악(`countDuplicateCharacters`) -> 빈도수가 높은 순서대로 정렬 -> 압축 코드 생성.

#### 4. 보안 및 패스워드 정책 검증
회원가입 시 비밀번호에 동일한 문자가 3번 이상 반복되지 않도록 제한하는 로직에 사용됩니다.
* **예시**: `Map<String, Long>`의 결과값(Value) 중에 3 이상의 숫자가 있는지 확인하여 사용자에게 경고를 보냅니다.
