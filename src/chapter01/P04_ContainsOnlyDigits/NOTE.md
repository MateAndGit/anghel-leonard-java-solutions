# [Java] Strings 클래스 분석: 숫자 포함 여부 체크 (ContainsOnlyDigits)

이 클래스는 입력된 문자열이 숫자로만 이루어졌는지 검사하는 세 가지 다른 접근 방식을 보여줍니다. 실무에서는 성능, 가독성, 유지보수 편의성에 따라 선택이 달라집니다.

---

## 1. containsOnlyDigitsV1: 루프 기반 검사
가장 전통적이고 성능 지향적인 방식입니다.

### [코드 분석]
- **동작 방식**: 문자열을 처음부터 끝까지 `for`문으로 돌며 `Character.isDigit()` 메서드로 각 문자가 숫자인지 확인합니다.
- **특징**: 숫자가 아닌 문자를 발견하는 즉시 `false`를 반환(Early Return)하므로 매우 효율적입니다.

### [실무 활용 사례]
- **고성능 시스템**: 대량의 사용자 입력 데이터를 실시간으로 검증해야 하는 백엔드 로직에서 가장 선호됩니다. 추가 객체 생성이 없어 가비지 컬렉션(GC) 부하가 가장 적습니다.

---

## 2. containsOnlyDigitsV2: 정규표현식(Regex) 방식
코드가 가장 짧고 명확하지만, 성능과 문법에 주의해야 합니다.

### [코드 분석]
- **동작 방식**: `String.matches()`를 사용하여 정규식 `[0-9]+`와 비교합니다.
- **주의사항**: 제시된 코드에 오타(`]`가 하나 더 있음)가 있습니다. 올바른 표현식은 `str.matches("[0-9]+")`입니다.
- **특징**: 가독성이 좋으나, 내부적으로 호출될 때마다 `Pattern` 객체를 컴파일하므로 반복 호출 시 성능이 떨어집니다.

### [실무 활용 사례]
- **복잡한 패턴 검증**: 단순히 숫자뿐만 아니라 "숫자 6자리", "특정 숫자로 시작하는 번호" 등 조건이 복잡해질 때 정규식으로 확장하여 사용합니다.

---

## 3. containsOnlyDigitsV3: Java Stream API 방식
Java 8 이상의 함수형 프로그래밍 스타일입니다.

### [코드 분석]
- **동작 방식**: `str.chars()`를 통해 정수 스트림을 생성하고, `anyMatch`를 이용하여 숫자가 아닌 것이 하나라도 있는지 검사한 뒤 결과를 반전(`!`)시킵니다.
- **특징**: `V1`의 성능과 `V2`의 간결함 사이의 절충안입니다.

### [실무 활용 사례]
- **선언적 프로그래밍**: 팀 내 코딩 컨벤션이 스트림 사용을 지향할 때 사용하며, 병렬 처리가 필요한 아주 긴 문자열 검사 시 `parallel()`과 조합하기 좋습니다.

## 5. 실무 적용 팁 및 참조
### 실무 유형별 추천 메서드

1. **사용자 회원가입 폼 검증 (UI Backend)**
    - **추천**: `containsOnlyDigitsV2 (Regex)`
    - **이유**: 속도보다는 정확성과 확장성이 중요합니다. 나중에 "6자리 숫자만"으로 요건이 바뀌면 `[0-9]{6}`으로 정규식만 수정하면 되기 때문입니다.

2. **금융 거래 대량 배치 처리 (Batch Job)**
    - **추천**: `containsOnlyDigitsV1 (Loop)`
    - **이유**: 수백만 건의 데이터를 처리할 때는 성능이 최우선입니다. `V1`은 객체 생성을 최소화하여 시스템 부하를 줄입니다.

3. **내부 관리자 도구 (Admin Tool)**
    - **추천**: `containsOnlyDigitsV3 (Stream)`
    - **이유**: 동료 개발자들이 코드를 읽기 편해야(Readability) 유지보수가 쉽습니다.
