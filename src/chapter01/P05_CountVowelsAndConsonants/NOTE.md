# [Java] Strings 클래스 분석: 모음(Vowels) & 자음(Consonants) 카운팅

이 클래스는 입력받은 문자열에서 영문 모음(a, e, i, o, u)과 그 외의 자음 개수를 파악하여 `Pair` 객체에 담아 반환합니다.

---

## 1. 클래스 내부 구조 분석

### ① 상수 및 설정
- `allVowels`: `HashSet`을 사용하여 모음 목록을 관리합니다. `List`보다 `contains()` 메서드의 속도가 훨씬 빠르기 때문에 효율적입니다.
- **대소문자 처리**: 로직 시작 전 `str.toLowerCase()`를 수행하여 비교 로직을 단순화했습니다.

### ② countVowelsAndConsonantsV1 (반복문 방식)
- **동작**: 문자열을 한 글자씩 순회하며 `if-else`로 모음인지, 혹은 'a~z' 사이의 자음인지 판별합니다.
- **특징**: 직관적이며, 추가적인 객체 생성(Stream 등)이 적어 메모리 효율이 좋습니다.

### ③ countVowelsAndConsonantsV2 (Stream & Partitioning 방식)
- **동작**: `partitioningBy()`라는 특수한 Collector를 사용합니다.
- **핵심 기술**:
   - `filter(ch >= 'a' && ch <= 'z')`: 특수문자나 숫자를 먼저 걸러냅니다.
   - `partitioningBy(조건, counting())`: 조건을 만족하는 그룹(모음)과 만족하지 않는 그룹(자음)으로 단 한 번의 스트림 순회로 데이터를 나눕니다.
- **결과 반환**: `result.get(true)`는 모음 개수, `result.get(false)`는 자음 개수가 됩니다.

## 2. 실무 활용 사례

이런 텍스트 분석 로직은 단순한 개수 세기를 넘어 다양한 분야에 응용됩니다.

### [사례 1: 자연어 처리(NLP) 및 텍스트 마이닝]
- 언어별 특징 추출: 영어와 타 언어(예: 루마니아어, 프랑스어)는 모음의 비율이 다릅니다. 텍스트의 모음/자음 비율을 분석하여 해당 문장이 어떤 언어인지 추측하는 모델의 기초 데이터로 활용합니다.

### [사례 2: 가독성 점수 계산 (Readability Score)]
- 텍스트의 난이도를 측정하는 알고리즘(예: Flesch-Kincaid)에서 음절(Syllable) 수를 계산할 때 모음의 개수와 배치를 기본 지표로 사용합니다.

### [사례 3: 보안 및 스팸 필터링]
- **난독화 탐지**: 임의로 생성된 스팸 텍스트나 암호화된 문자열은 일반적인 단어에 비해 자음만 연속되거나 모음 비율이 비정상적인 경우가 많습니다. 이를 통계적으로 걸러내는 필터링 로직에 응용됩니다.

## 3. 실무형 요약 보고서

| 특징 | V1 (Loop) | V2 (Stream & Partitioning) |
| :--- | :--- | :--- |
| **성능** | **매우 빠름** (오버헤드 없음) | 보통 (스트림/박싱 비용 발생) |
| **코드 가독성** | 절차적이고 명시적임 | **매우 높음** (함수형 스타일) |
| **확장성** | 조건 추가 시 `if`문이 복잡해짐 | `Predicate`만 수정하면 되어 관리가 쉬움 |
| **특이사항** | 마지막에 `Pair.of()`로 결과 결합 | `partitioningBy`로 한 번에 그룹화 |

## 4. 현업 개발자의 유지보수 팁

1. **유니코드 주의**: 주석에도 있듯이 `charAt()`은 2바이트를 넘어가는 이모지나 특수 유니코드를 처리하지 못합니다. 글로벌 서비스라면 `codePoints()`를 사용하는 것이 안전합니다.
2. **다국어 확장**: 실무에서는 모음 셋(`allVowels`)을 외부 설정 파일(YAML, JSON)이나 DB로 분리하여, 언어별로 동적으로 로딩하게 설계하는 경우가 많습니다.
3. **정규식 대안**: 간단한 확인이라면 `str.replaceAll("(?i)[^aeiou]", "").length()` 같은 정규식 방식도 있지만, 성능상 위 코드들이 훨씬 유리합니다.
