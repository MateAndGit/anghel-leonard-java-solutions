# 🔍 Strings 클래스 분석: 반복되지 않는 첫 번째 문자 찾기

이 코드는 문자열을 훑으며 **등장 횟수가 정확히 1인 문자 중 가장 앞에 있는 것**을 찾는 5가지 방법을 보여줍니다.

---

### 1. V1: 이중 for 루프 (Brute Force)
가장 직관적이지만 가장 느린 방식입니다.
* **방법**: 첫 번째 문자를 잡고, 나머지 전체 문자열을 다 뒤져서 똑같은 게 있는지 확인합니다. 없으면 바로 반환!
* **성능**: 문자열 길이가 $n$일 때 $O(n^2)$의 시간이 걸려, 문자열이 길어지면 매우 느려집니다.

---

### 2. V2: 배열 활용 (ASCII 전용)
고정된 크기의 배열을 일종의 "체크리스트"로 사용합니다.
* **방법**: 256칸짜리 배열(`flags`)을 만들어서 문자의 위치를 기록합니다.
  * 처음 보면: 인덱스(위치) 저장
  * 두 번 이상 보면: `-2`로 표시해서 제외
* **특징**: `char`를 숫자로 바로 배열 인덱스에 넣기 때문에 매우 빠르지만, 유니코드(이모지 등) 대응이 어렵습니다.

---

### 3. V3: LinkedHashMap 활용 (정석)
실무에서 가장 많이 권장되는 균형 잡힌 방식입니다.
* **핵심**: 일반 `HashMap`은 순서를 보장하지 않지만, **`LinkedHashMap`은 입력된 순서를 기억**합니다.
* **방법**:
  1. 문자열을 한 번 읽으며 각 문자의 개수를 맵에 저장합니다.
  2. 맵을 순회하며 개수가 `1`인 첫 번째 Key를 반환합니다. 순서가 보장되므로 무조건 "첫 번째" 문자가 나옵니다.

---

### 4. V4: Stream API (현대적 자바)
V3의 로직을 자바 8 이상의 Stream 문법으로 깔끔하게 한 줄로 구현한 것입니다.
* **특징**: `Collectors.groupingBy`를 쓸 때 `LinkedHashMap::new`를 인자로 넘겨서 순서를 유지하는 것이 핵심입니다.

---

### 5. VCP4: Code Point 대응 (완벽한 유니코드)
앞서 배운 것처럼 이모지나 특수한 유니코드를 처리할 수 있는 버전입니다.
* **차이점**: `str.chars()` 대신 `str.codePoints()`를 사용하여 '😊' 같은 보충 문자도 하나의 문자로 정확히 인식합니다.

---

### 📊 방식별 성능 및 특징 비교

| 버전 | 시간 복잡도 | 공간 복잡도 | 특징 |
| :--- | :--- | :--- | :--- |
| **V1** | $O(n^2)$ | $O(1)$ | 코드가 쉽지만 대용량 데이터에 쥐약 |
| **V2** | $O(n)$ | $O(1)$ | 매우 빠르나 ASCII 환경에서만 안전 |
| **V3/V4** | $O(n)$ | $O(n)$ | **가장 추천.** 순서 보장 + 모든 문자 대응 |
| **VCP4** | $O(n)$ | $O(n)$ | **최종 보스.** 이모지 포함 전 세계 문자 대응 가능 |

### 🏢 "반복되지 않는 첫 번째 요소 찾기"의 실무 활용 사례

#### 1. 로그 분석 및 에러 디버깅 (Unique Event Finding)
수만 개의 시스템 로그 중에서 **단 한 번만 발생한 특이 에러(Unique Error)**를 찾아내야 할 때 이 로직을 사용합니다.
* **상황**: 수천 번 발생하는 일반적인 경고(Warning)들 사이에서, 시스템 장애의 근본 원인이 되는 딱 한 번의 'Unique Event'를 시간 순서대로 찾아야 할 때.
* **적용**: 로그 메시지를 키(Key)로 삼아 빈도수를 계산한 후, 개수가 1인 첫 번째 로그를 반환합니다.

#### 2. 실시간 스트리밍 데이터의 "최초 유입자" 식별
채팅방이나 실시간 이벤트 페이지에서 중복 참여를 제외하고, **정말로 처음으로 액션을 취한 유효한 사용자**를 가려낼 때 사용합니다.
* **상황**: 퀴즈 정답자 선착순 이벤트에서 한 사람이 여러 번 정답을 외쳤을 때, 중복된 사람들을 제외하고 가장 먼저 정답을 한 번만 말한 사람을 선발.

#### 3. 보안 및 인증 (One-Time Key 검증)
패킷 데이터나 통신 데이터에서 재전송 공격(Replay Attack)을 방지하기 위해, **중복되지 않은 식별자(Nonce)** 중 가장 먼저 도착한 정상 패킷을 구분할 때 응용됩니다.

#### 4. UI/UX: 검색어 하이라이팅 및 자동 완성 우선순위
사용자가 입력한 검색어 중 문맥상 가장 중요한(중복되지 않은) 키워드를 추출하여 UI에 먼저 보여줄 때 사용합니다.

### 🛠️ 실무급 코드 응용 (V4/VCP4 방식 응용)

실무에서는 단순히 `char`를 찾는 게 아니라, 특정 **객체(Object)**에서 중복을 체크하는 식으로 응용합니다.

```java
// 실무 예시: 로그 리스트에서 중복되지 않은 첫 번째 에러 객체 찾기
public LogEntry findFirstUniqueError(List<LogEntry> logs) {
    return logs.stream()
        .collect(Collectors.groupingBy(LogEntry::getErrorMessage, 
                  LinkedHashMap::new, Collectors.counting())) // 메시지별로 개수 세기
        .entrySet().stream()
        .filter(entry -> entry.getValue() == 1)               // 한 번만 발생한 것 필터
        .findFirst()                                          // 그중 첫 번째
        .map(entry -> findLogByMessage(logs, entry.getKey())) // 해당 로그 객체 반환
        .orElse(null);
}
