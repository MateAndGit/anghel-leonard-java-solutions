# [Java] Strings 클래스 분석: 아나그램(Anagram) 판별

두 문자열이 순서만 다르고 구성 문자가 동일한지 확인하는 세 가지 접근 방식을 다룹니다. 성능 최적화와 모던 자바 문법의 조화를 배우기에 좋습니다.

---

## 1. 메서드별 상세 분석

### ① V1: 정렬(Sorting) 방식
- **원리**: 두 문자열을 모두 소문자로 바꾸고 공백을 제거한 뒤, **정렬(Sort)**합니다. 정렬된 두 배열이 같으면 아나그램입니다.
- **특징**: 직관적이고 코드가 깔끔합니다. 정렬 알고리즘의 시간 복잡도($O(n \log n)$)를 따릅니다.

### ② V2: 카운팅(Counting) 방식 (최적 성능)
- **원리**: 크기가 256인 정수 배열을 준비합니다. 첫 번째 문자열의 문자가 나오면 `+1`, 두 번째 문자열의 문자가 나오면 `-1`을 합니다.
- **특징**: 모든 인덱스가 0이면 아나그램입니다. 정렬을 하지 않고 한 번의 루프($O(n)$)로 끝나기 때문에 **가장 빠릅니다.**

### ③ V3: Stream API 방식
- **원리**: `str1.chars().sorted()`를 사용하여 스트림 수준에서 정렬하고 비교합니다.
- **특징**: 별도의 `char[]` 변수를 선언하지 않아도 되며, 모던 자바 스타일의 가독성을 제공합니다.

---

## 2. 알고리즘 시각화: 카운팅 방식의 원리

카운팅 방식은 마치 **양팔 저울**에 문자를 하나씩 올리고 내리는 과정과 같습니다. 모든 처리가 끝났을 때 저울이 수평(0)을 이루면 아나그램입니다.



---

## 3. 실무 활용 사례

### [사례 1: 암호학 및 텍스트 변조 탐지]
- 특정 문장이 재배치된 데이터인지 확인하거나, 간단한 치환 암호문에서 원문과의 문자 빈도수를 대조할 때 기초 로직으로 사용됩니다.

### [사례 2: 검색 엔진의 유사어 추천]
- 사용자가 오타를 입력했을 때, 사전에 있는 단어 중 구성 문자가 비슷한(아나그램 관계인) 단어를 추천 엔진의 후보군으로 올릴 수 있습니다.

### [사례 3: 코딩 테스트 단골 문제]
- `HashMap`을 활용할 것인지, `int[]` 배열을 활용할 것인지에 대한 성능 판단 능력을 평가할 때 핵심 예제로 활용됩니다.

---

## 4. 실무형 요약 보고서

| 방식 | 시간 복잡도 | 공간 복잡도 | 추천 상황 |
| :--- | :---: | :---: | :--- |
| **Sort (V1)** | $O(n \log n)$ | $O(n)$ | 구현 속도가 중요할 때 |
| **Counting (V2)** | **$O(n)$** | $O(1)$ | **대용량 데이터 처리 및 고성능 요구 시** |
| **Stream (V3)** | $O(n \log n)$ | $O(n)$ | 간결한 코드 스타일 선호 시 |

## 5. 현업 개발자의 유지보수 팁 📝

### ⚠️ 전처리의 중요성
코드에 포함된 `.replaceAll("\\s", "").toLowerCase()`는 매우 중요합니다. 실무 데이터는 공백이나 대소문자가 제각각일 수 있기 때문에, 비교 전 **데이터 정규화(Normalization)** 과정이 필수적입니다.

### 💡 유니코드(Unicode) 주의
현재 `V2`는 ASCII 범위(256)만 지원합니다. 만약 한글이나 이모지 등이 섞인 아나그램을 판별해야 한다면 `int[]` 배열 대신 `HashMap<Integer, Integer>`를 사용하여 CodePoint를 카운팅해야 합니다.
