# [Java] Strings 클래스 분석: 특정 문자 제거 (Remove Given Character)

입력받은 문자열에서 특정 문자 하나를 완전히 삭제하는 네 가지 방식을 비교합니다.

---

## 1. 메서드별 상세 분석

### ① V1: StringBuilder & Loop 방식
- **동작**: 문자열을 한 글자씩 돌며 제거할 문자와 다를 경우에만 `StringBuilder`에 추가합니다.
- **특징**: 가장 원초적이고 직관적입니다. 추가 객체 생성이 적어 성능 면에서 준수합니다.

### ② V2: replaceAll() 정규식 방식
- **동작**: `replaceAll`을 사용하여 일치하는 문자를 빈 문자열("")로 바꿉니다.
- **특징**: 코드가 가장 짧습니다. `Pattern.quote()`를 사용하여 제거할 문자가 특수문자(예: `.`, `*`)일 경우 정규식으로 오해받지 않도록 안전하게 처리했습니다.

### ③ V3: Java Stream 방식
- **동작**: `chars()` 스트림을 열어 `filter`로 해당 문자를 걸러낸 뒤 다시 합칩니다.
- **특징**: 가독성이 좋지만, 문자를 하나씩 `String` 객체로 변환하는 과정에서 오버헤드가 발생할 수 있습니다.

### ④ V4: CodePoint (Unicode) 대응 방식 (추천)
- **동작**: `char` 대신 `codePoint`를 기준으로 필터링합니다.
- **특징**: 2바이트를 점유하는 이모지나 특수 유니코드를 완벽하게 식별하고 제거할 수 있는 가장 **강력한(Robust)** 방식입니다.

## 3. 실무 활용 사례

### [사례 1: 전화번호나 계좌번호의 하이픈(-) 제거]
- "010-1234-5678"에서 `-`를 제거하여 숫자만 남긴 데이터로 DB에 저장할 때 사용합니다.
- `removeCharacterV1(phoneNum, '-')`

### [사례 2: 불필요한 특수문자 제거]
- 텍스트 분석 전, 마침표(`.`)나 쉼표(`,`) 등을 일괄적으로 제거하여 단어의 원형만 남길 때 유용합니다.

### [사례 3: 보안 및 금지어 처리]
- 특정 시스템 예약어나 프로그래밍 기호를 사용자의 입력값에서 강제로 제거하여 보안 사고(Injection)를 방지하는 1차적인 수단으로 쓰입니다.

## 5. 현업 개발자의 유지보수 팁 📝

### ⚠️ 왜 `Pattern.quote()`를 썼을까요?
`replaceAll`의 첫 번째 인자는 **정규표현식**입니다. 만약 마침표(`.`)를 지우고 싶어서 그냥 넣으면, 정규식에서 `.`은 "모든 문자"를 의미하므로 **문자열 전체가 지워지는 대참사**가 발생합니다. `Pattern.quote()`는 이런 사고를 막아주는 보호막입니다.

### 💡 String.replace() vs replaceAll()
단순히 문자 하나를 지울 때는 정규식을 쓰는 `replaceAll`보다 `str.replace(String.valueOf(ch), "")`를 쓰는 것이 더 빠르고 안전합니다.
