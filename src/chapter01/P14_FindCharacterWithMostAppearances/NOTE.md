# [Java] Strings 클래스 분석: 최다 빈도 문자 찾기 (Most Appearances)

입력된 문자열에서 공백을 제외하고 가장 많이 나타난 문자가 무엇인지, 그리고 몇 번 나타났는지 찾아내는 세 가지 방식을 비교합니다.

---

## 1. 메서드별 상세 분석

### ① V1: HashMap 방식
- **원리**: `HashMap<Character, Integer>`를 생성하여 문자를 키(Key)로, 등장 횟수를 값(Value)으로 저장합니다.
- **특징**: 어떤 유니코드 문자가 들어와도 유연하게 대응할 수 있습니다. `Collections.max()`를 사용해 최댓값을 먼저 찾고 다시 맵을 돌며 해당 문자를 찾는 과정을 거칩니다.

### ② V2: ASCII 배열 방식 (성능 최적화)
- **원리**: 크기가 256인 `int[]` 배열을 미리 만들고, 문자의 ASCII 코드를 인덱스로 활용하여 카운트를 올립니다.
- **특징**: `HashMap` 같은 객체를 생성하지 않고 배열(Primitive)만 사용하므로 **속도가 매우 빠릅니다.** 다만, 확장 ASCII(256자) 범위를 벗어나는 유니코드에는 대응하기 어렵습니다.

### ③ V3: Java Stream & groupingBy 방식
- **원리**: 스트림을 통해 문자를 그룹화(`groupingBy`)하고 개수를 세는(`counting`) 함수형 접근 방식입니다.
- **특징**: 코드가 매우 우아하며, `max(comparingByValue())`를 통해 한 번의 흐름으로 최다 빈도 문자를 추출합니다. 실무에서 가장 가독성이 좋은 코드입니다.

---

## 2. 알고리즘 시각화: 빈도수 카운팅 (Frequency Counting)

최다 빈도 문자를 찾는 과정은 각 항목의 개수를 센 뒤 가장 높은 막대그래프를 고르는 것과 같습니다.



---

## 3. 실무 활용 사례

### [사례 1: 텍스트 데이터 마이닝]
- 특정 문서에서 가장 자주 사용된 단어나 철자를 분석하여 텍스트의 성격(주제)을 파악하거나, 작가의 글쓰기 습관을 연구할 때 기초 로직으로 쓰입니다.

### [사례 2: 압축 알고리즘 (Huffman Coding)]
- 파일을 압축할 때 가장 많이 등장하는 문자에 짧은 비트를 할당하기 위해 데이터의 빈도수를 먼저 계산해야 합니다. 이때 이와 같은 카운팅 로직이 필수적입니다.

### [사례 3: 보안 및 이상 탐지]
- 네트워크 패킷이나 로그에서 특정 문자나 패턴이 비정상적으로 많이 등장하는 것을 감지하여 DDoS 공격이나 시스템 에러를 모니터링할 때 사용합니다.

## 4. 실무형 요약 보고서

| 방식 | 가독성 | 성능 | 특징 | 추천 상황 |
| :--- | :---: | :---: | :--- | :--- |
| **HashMap (V1)** | 보통 | 좋음 | 모든 문자 대응 가능 | 일반적인 비즈니스 로직 |
| **Array (V2)** | 보통 | **최상** | 메모리 오버헤드 최소 | **대량의 영문 텍스트 처리 시** |
| **Stream (V3)** | **최상** | 보통 | 모던 자바 스타일 | 가독성과 유지보수 중시 |

## 5. 현업 개발자의 유지보수 팁 📝

### ⚠️ 공백 처리 (Whitespace)
위의 모든 코드에는 `!Character.isWhitespace(currentCh)` 조건이 들어있어 공백(띄어쓰기, 탭 등)은 무시합니다. 만약 공백이 가장 많은 문자인지 확인해야 하는 기획 요건이 있다면 이 필터를 제거해야 합니다.

### ⚠️ 최다 빈도가 여러 개일 경우
현재 로직은 가장 마지막에 발견된 최다 빈도 문자 하나만 반환합니다. 만약 "AAABBB"처럼 A와 B의 횟수가 같을 때 둘 다 반환해야 한다면, 결과 타입을 `Pair<List<Character>, Integer>`와 같은 형태로 확장해야 합니다.
