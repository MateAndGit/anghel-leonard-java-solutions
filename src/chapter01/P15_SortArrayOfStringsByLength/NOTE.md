# [Java] Strings 클래스 분석: 문자열 배열 길이순 정렬 (Sort by Length)

이 예제는 Java에서 객체를 정렬할 때 사용하는 `Comparator`의 진화 과정을 보여줍니다. 람다식에서부터 최신 메서드 참조 방식까지 단계별로 학습할 수 있습니다.

---

## 1. 메서드별 상세 분석

### ① V1: 람다(Lambda)와 Integer.compare
- **원리**: `Arrays.sort`에 두 문자열의 길이를 비교하는 람다식을 전달합니다.
- **특징**: 내림차순을 구현하기 위해 결과값에 `-1`을 곱하는 고전적인 방식을 보여줍니다. 정렬의 기본 원리를 이해하기 좋습니다.

### ② V2: Comparator.comparingInt (추천)
- **원리**: Java 8에서 도입된 `Comparator`의 정적 메서드를 사용합니다.
- **특징**: `String::length`라는 메서드 참조를 사용하여 코드가 매우 읽기 쉽습니다. `.reversed()`만 붙여주면 내림차순이 즉시 구현되므로 **실무에서 가장 많이 쓰이는 방식**입니다.

### ③ V3: Stream API 방식
- **원리**: 원본 배열을 건드리지 않고, 정렬된 새로운 배열을 생성하여 반환합니다.
- **특징**: "원본 데이터는 변하지 않아야 한다(Immutability)"는 함수형 프로그래밍 원칙을 지킬 때 유용합니다.

## 3. 실무 활용 사례

### [사례 1: 연관 검색어 및 자동 완성]
- 검색창에 단어를 입력했을 때, 짧은 단어부터 혹은 긴 단어부터 우선순위를 두어 사용자에게 노출해야 하는 UI 로직에 사용됩니다.

### [사례 2: 텍스트 시각화 및 레이아웃]
- 리포트나 대시보드에서 문자열의 길이에 따라 컴포넌트의 크기를 조절하거나, 시각적으로 균형 잡힌 목록을 보여주기 위해 데이터를 미리 정렬할 때 필요합니다.

### [사례 3: 자연어 처리(NLP) 전처리]
- 불용어(Stopwords)를 제거하거나 단어 사전을 만들 때, 특정 길이 이하의 짧은 단어들을 걸러내기 전 단계로 정렬을 수행하기도 합니다.

## 5. 현업 개발자의 유지보수 팁 📝

### ⚠️ Null 안전성
실무 데이터에는 배열 안에 `null`이 섞여 있을 수 있습니다. `s1.length()`를 호출하는 순간 `NullPointerException`이 발생하므로, 데이터가 불안정하다면 `Comparator.nullsLast()` 등을 조합하여 방어 코드를 짜야 합니다.

### 💡 Stable Sort (안정 정렬)
`Arrays.sort()`는 **안정 정렬**입니다. 즉, 길이가 같은 문자열끼리는 원래 있던 순서가 유지됩니다. 만약 길이가 같을 때 2차로 사전순 정렬을 하고 싶다면 아래와 같이 체이닝할 수 있습니다.
```java
Comparator.comparingInt(String::length).thenComparing(Comparator.naturalOrder())
