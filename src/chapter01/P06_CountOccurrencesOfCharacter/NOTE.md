# [Java] Strings 클래스 분석: 특정 문자 출현 횟수 세기

이 클래스는 문자열 내 특정 문자의 개수를 계산합니다. 일반적인 `char` 처리 방식과 유니코드 대리 쌍(Surrogate Pairs)을 고려한 `CodePoint` 처리 방식의 차이를 배우는 것이 핵심입니다.

---

## 1. 메서드별 상세 분석

### ① V1 시리즈: replace()를 이용한 트릭
- **원리**: 전체 문자열 길이에서 "찾으려는 문자를 빈 문자열("")로 치환한 문자열"의 길이를 뺍니다.
- **특징**: 반복문을 직접 돌지 않아 코드가 간결합니다.
- **VCP1 (Unicode 대응)**: 유니코드 보충 문자는 길이가 2(char 2개 점유)이므로, 결과값을 2로 나누어 보정하는 디테일이 포함되어 있습니다.

### ② V2 시리즈: 전통적인 For-loop 방식
- **원리**: 문자열을 처음부터 끝까지 순회하며 비교합니다.
- **특징**: `charAt()`을 쓰는 방식과 유니코드 대응을 위해 `codePointAt()`을 쓰는 방식으로 나뉩니다.
- **VCP2**: 2바이트가 넘는 특수 기호나 이모지까지 정확하게 카운트할 수 있는 가장 정석적인 로직입니다.

### ③ V3 시리즈: Stream API 방식
- **원리**: `chars()` 또는 `codePoints()`를 스트림으로 변환하여 `filter`로 개수를 셉니다.
- **특징**: 모던 자바 스타일이며 코드가 매우 읽기 쉽습니다.

---

## 2. 실무 활용 사례

문자열 검색과 필터링은 실무에서 다음과 같이 응용됩니다.

### [사례 1: CSV 또는 구분자 기반 데이터 파싱]
데이터 파일(CSV)을 읽을 때 한 줄에 쉼표(`,`)가 몇 개 있는지 확인하여 데이터의 형식이 올바른지(컬럼 개수 검증) 1차 체크할 때 사용합니다.
- 예: `countOccurrences(line, ',')`

### [사례 2: 소스 코드 및 텍스트 분석]
특정 프로그래밍 언어나 텍스트에서 괄호 `(` `)`의 개수가 맞는지 확인하거나, 특정 키워드가 몇 번 언급되었는지 빈도 분석을 할 때 기본 로직으로 쓰입니다.

### [사례 3: 글로벌 서비스의 이모지/특수기호 처리]
SNS 서비스에서 사용자가 입력한 게시물에 특정 이모지(예: ❤️)가 몇 개 포함되었는지 통계를 낼 때, `VCP` 방식(CodePoint)을 사용해야 정확한 개수가 집계됩니다.

---

## 3. 실무형 요약 보고서

| 방식 | 가독성 | 성능 | 유니코드 대응 | 추천 상황 |
| :--- | :---: | :---: | :---: | :--- |
| **replace (V1)** | 상 | 낮음 | 가능 (나눗셈 보정) | 코드를 짧게 짜고 싶을 때 |
| **Loop (V2)** | 보통 | **최상** | **VCP2 강력 추천** | 대용량 텍스트 처리 시 |
| **Stream (V3)** | **최상** | 보통 | 가능 | 유지보수와 가독성이 중요할 때 |

---

## 4. 현업 개발자의 메모 📝

### ⚠️ 왜 `VCP` (CodePoint) 방식이 중요한가요?
우리가 흔히 쓰는 이모지나 고대 문자 등은 내부적으로 `char` 2개를 합쳐서 하나의 문자를 표현합니다(Surrogate Pair).
- 단순 `char` 방식: 이모지 하나를 2개로 인식하는 오류 발생.
- `CodePoint` 방식: 하나의 이모지를 정확히 1개로 인식.



### 실무 팁
대부분의 상황에서는 **V3(Stream)** 방식이 가장 무난하지만, 성능이 정말 중요한 루프 안에서는 **V2(Loop)** 방식을 권장합니다. 유니코드가 섞일 가능성이 있다면 반드시 `codePoints()` 계열을 사용하세요!
